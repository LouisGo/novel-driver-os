# 商业叙事协议

处理题材、爽点、节奏、章节 brief、样本学习或章节审稿时，必须把“套路”拆成可验证的叙事机制，而不是套模板。

## 核心原则

- 套路不是问题；问题是无新意、无包装、无人物代价。
- 爽点必须符合人性：被轻视后的证明、被剥夺后的夺回、不公秩序的反转、能力被验证、关系被承认、选择产生回报。
- 节奏必须有波形：压力、蓄势、兑现、缓冲、余波、转向、下一轮期待。
- 题材要有碰撞：熟悉母题加陌生变量、现实情绪、反差职业、特殊代价、意想不到的世界规则。
- 行文不能暴露方法论：不要掉书袋、不要写成设定论文、不要让人物像功能模块。

## 必查项

- `freshness_hook`：一句话卖点是否有“熟悉但没见过”的反差。
- `human_need`：读者爽感对应哪种人性需求。
- `cost_and_consequence`：主角收获是否付出代价，并改变后续局面。
- `promise_delay_payoff`：承诺、延迟、兑现的间隔是否清楚。
- `emotion_curve`：本章或本段是否避免持续平路、持续开挂或持续受虐。
- `naturalization`：设定、主题和技巧是否被场景、动作、关系自然承载。

## 禁止

- 不把“热血、燃、爽、史诗感”当作分析结论。
- 不为了新意牺牲主角代入和读者理解。
- 不用宏大主题压过具体人物选择。
- 不把通用创作理论直接写进正文。
