# 文风系统参考

用于 `novel-style-evolution`。

## 文件职责

- `style_bible.md`：只放作者确认的稳定规则。
- `aspirational_style.md`：作者想靠近的方向，不等于模仿。
- `anti_style.md`：明确不要的表达、节奏、桥段、AI 味。
- `discarded_brilliance.md`：废案和未来复活资产。
- `style_entropy_budget.md`：保留实验空间，防止文风锁死。

## 升级规则

- 文风候选 不能直接进入 style_bible。
- 至少需要作者确认或多次高权威输入支持。
- 每条规则都要有“适用范围”和“过度使用风险”。
- 单章审稿低分只能进入 `style_candidates.md` 或 `anti_style.md` 候选，不能直接固化。
- 只有连续多章出现同类问题，或作者明确确认，才能把“人味/AI 味/钩子/节奏”问题升级为长期规则。

## 中文网文维度

- 追读节奏。
- 爽点密度。
- 章末钩子强度。
- 对白信息量。
- 人味细节：细节必须塑造人物、处境、欲望或关系。
- 主角锚定：开头和关键场景优先围绕主角压力、判断和行动。
- AI 味风险。
- 关系线表达方式。

## 质量审稿到文风规则

从 `chapter_quality_review` 提炼规则时，按下列映射：

- `protagonist_anchor` 低分 -> aspirational：开头先压到主角身上；anti-style：先讲世界史。
- `hook_ladder` 低分 -> aspirational：每章结尾提出具体下一章问题；anti-style：空泛预告。
- `human_texture` 低分 -> aspirational：细节服务人物；anti-style：素材库式描写。
- `ai_flavor` 低分 -> anti-style：均匀段落、机械过渡、抽象总结。
- `payoff` 低分 -> aspirational：每章至少一次局部回报；anti-style：只有铺垫无兑现。
