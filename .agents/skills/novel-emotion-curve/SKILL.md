# 情绪曲线

## 职责

像设计波形一样设计读者情绪：让压力、惊喜、兑现、缓冲和下一轮期待有节奏地出现。

## 必读

- 目标章节、桥段、大纲或最近章节质量趋势。
- 当前卷目标、上一章结尾、本章预期钩子。
- `../_shared/author-facing-language-protocol.md`。
- `../_shared/commercial-storycraft-protocol.md`。
- 如审查正文，读 `../novel-human-chapter-intake/references/chapter-quality-rubric.md`。

## 曲线单位

- `pressure`：风险、羞辱、失去、限制、倒计时。
- `build`：信息延迟、选择收窄、代价加重。
- `payoff`：小胜、验证、反转、关系确认。
- `breath`：大战后余波、身体感、关系反应、局面清点。
- `turn`：新问题、新代价、新敌意、新诱惑。

## 工作流程

1. 标出当前文本或计划的情绪节点，不要只总结剧情。
2. 检查是否连续开挂、连续受挫、连续解释、连续平路。
3. 给出本章或本段最小曲线：压力从哪里来，何处蓄势，何处兑现，何处喘息，结尾如何转向。
4. 大战或高强度桥段后必须设计余波；长期压抑后必须给可信小回报。
5. 输出曲线报告或下一章节奏建议。

## 输出

面向作者时输出“情绪节奏草案”或“情绪节奏审查”，不要把 `emotion_curve_report` 当标题。必须使用作者态结构：

- `【类型】情绪节奏草案` 或 `【类型】情绪节奏审查`
- `【状态】未确认`
- `【入库】不会写入正史资料，需要你确认后才可进入长期资料`
- `【用途】用于安排压力、蓄势、兑现、缓冲和章末转向`
- `【一句话锚点】`：先说明本段或本章的情绪波形。
- `【当前问题】`：如连续解释、持续受虐、持续开挂或平路。
- `【推荐曲线】`：按 `【压力】`、`【蓄势】`、`【兑现】`、`【缓冲】`、`【转向】` 展开。
- `【读者感受】`：说明每个节点希望读者产生的情绪。
- `【章末钩子】`：说明结尾把期待转向哪里。
- `【需要你确认】`：列出不超过 5 个会改变节奏写法的开关。
- `【下一步】`：指向爽点设计、章节作战简报或改稿动作。

内部登记时可保存 `emotion_curve_report`：

- `source_input_id`
- `target_scope`
- `current_curve`
- `curve_risks`
- `recommended_curve`
- `breath_points`
- `surprise_points`
- `payoff_points`
- `turning_hook`
- `next_commands`

## 禁止

- 不把情绪曲线写成三幕式教科书。
- 不要求每章都大高潮。
- 不用空泛“起承转合”替代具体读者情绪。
- 不直接写正文。

## 自检

- 是否指出读者每一段应感到什么？
- 是否避免持续高潮、持续打击或一潭死水？
- 是否安排了大战后的缓冲或低谷后的回报？
- 章末是否把情绪转向下一章期待？
