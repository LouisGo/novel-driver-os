# 世界观契约

## 职责

把世界观灵感整理成可审查的规则草案。它服务“先把世界定下来”，不直接生成 memory patch，不把设定写入正史资料。

## 必读

- 作者原始世界观输入、题材方向、主角信息和禁区。
- 如已有项目：`project.yaml`、`10_bible/world_contract.md`、`10_bible/open_questions.md`、`30_plot/unresolved_hooks.md`。
- `../_shared/author-facing-language-protocol.md`。
- `../_shared/commercial-storycraft-protocol.md`。
- `../_shared/canon-safety-protocol.md`。

## 工作流程

1. 先抽出世界核心：这个世界最重要的异常规则是什么，为什么能支撑长篇。
2. 固定底层规则：列 3-5 条不可违背的规则，避免只堆名词。
3. 设计时代结构：旧秩序、当前压力、资源分配和主要矛盾。
4. 设计力量体系：力量来源、升级方式、代价、限制和可验证场景。
5. 设计势力格局：谁掌握资源，谁被排斥，谁提供灰色机会。
6. 找主角切口：主角为什么会卷入世界矛盾，凭什么能改变局面。
7. 给出长篇引擎：每卷如何自然产生地图、冲突、升级、爽点和新问题。
8. 输出待确认开关和风险；需要落盘时再转 `novel-memory-patch`。

## 输出

面向作者时输出“世界观草案”，必须使用作者态结构：

- `【类型】世界观草案`
- `【状态】未确认`
- `【入库】不会写入正史资料，需要你确认后才可进入长期资料`
- `【用途】用于确定底层规则、力量体系、势力格局和长篇引擎`
- `【一句话锚点】`：一句话说明世界为什么成立。
- `【世界核心】`：说明这个世界的最大异常和读者一眼能懂的吸引点。
- `【底层规则】`：列 3-5 条不可违背的规则，每条说明叙事用途。
- `【时代结构】`：旧秩序、当前压力、主要矛盾。
- `【力量体系】`：力量来源、升级方式、代价、限制。
- `【势力格局】`：资源掌控者、被压迫者、灰色空间和危险禁区。
- `【主角切口】`：主角的出身、压力、特殊能力或特殊位置。
- `【长篇引擎】`：卷级目标如何持续产生。
- `【需要你确认】`：不超过 5 个会改变世界写法的开关。
- `【风险】`：设定过密、主角旁观、爽点弱、科技玄幻割裂等风险。
- `【下一步】`：确认后转世界观补丁、创作方向、爽点设计或章节简报。

内部登记时可保存以下字段：

- `source_input_id`
- `world_core`
- `world_rules`
- `era_structure`
- `power_system`
- `faction_map`
- `protagonist_entry`
- `longform_engine`
- `confirmation_switches`
- `risks`
- `next_commands`

## 禁止

- 不把世界观草案写入 `world_contract.md` 或 `canon_registry.md`。
- 不用名词堆砌代替规则和矛盾。
- 不让世界观压过主角处境。
- 不把科技元素和玄幻元素并列摆放，必须说明它们如何互相解释。
- 不写正文。

## 自检

- 世界核心是否一句话可懂？
- 底层规则是否能制造剧情选择、代价和爽点？
- 主角是否有明确切入世界矛盾的位置？
- 长篇引擎是否能自然生成每卷目标？
- 是否明确了草案状态、入库边界和作者确认开关？
